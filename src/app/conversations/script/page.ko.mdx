# 메신저 기능 조작하기

메신저는 "도움말 요청" 버튼 클릭과 같은 사용자 상호작용 시 프로그래밍 방식으로 실행할 수 있습니다. 단일 페이지 애플리케이션에서는 모든 페이지에 런처 아이콘을 표시하는 것이 이상적이지 않기 때문에 문서에 첨부하는 대신 특정 요소에 erxes 런처를 첨부하는 옵션이 있어야 합니다. 이제 메신저의 스타일, 위치를 변경하고 지정된 페이지에서 메신저를 호출하기 위해 웹사이트에 버튼을 설정할 수 있습니다.

**버튼 제출**
외부(부모 웹사이트)에서 제출을 호출할 수 있는 기능으로, 외부에서 callSubmit 액션을 들어서 제출 액션을 강제로 실행할 수 있습니다. 예를 들어, 메신저를 열기 위한 액션을 호출하는 버튼을 추가할 수 있습니다. window 객체에 window.Erxes.showMessenger() 함수가 있습니다. 이 기능을 사용하여 메신저를 프로그래밍 방식으로 표시할 수 있습니다. 다음 스크립트를 메신저 스크립트 코드에 삽입하세요.

```tsx
 document.getElementById('button').onclick = () => {
    window.Erxes.showMessenger()
  }
  ```
기본 핸들러를 숨기기 위해 show launcher 체크를 해제해야 합니다. show launcher를 체크하면 위젯 섹션은 보이지 않지만 여전히 callSubmit 액션을 호출하여 메신저를 표시할 수 있습니다.

# 구글 태그 관리자 문제 해결 가이드
특정 페이지에 구글 태그 관리자 스크립트를 설치하는 데 문제가 있는 경우 설정이 올바른지 확인하는 몇 가지 문제 해결 단계를 수행할 수 있습니다. 이 지침은 **모든 웹 페이지에 메신저가 포함되어 있고 일부 특정 페이지에만 팝업 스크립트가 포함된 경우**를 다룹니다. 따라서 Erxes 스크립트를 변경하여 다른 경우에도 작동할 수 있습니다. 다음 지침은 웹 페이지에 Erxes 스크립트를 배열하는 방법을 보여줍니다.

페이지에 두 개의 스크립트 코드를 설치하려면 구글 태그 관리자 HTML 코드에 일부 트리거를 설정해야 합니다.

### 메신저 스크립트에서 트리거 구성:
1. 구글 태그 관리자에서 이미 구성한 HTML의 메신저 스크립트 코드에 트리거를 추가합니다.
2. 지정된 페이지를 포함하지 않는 메신저 스크립트의 트리거를 구성합니다. 이 경우 메신저 스크립트는 다른 태그에 구성됩니다. 다음 단락에서 메신저 + 팝업 스크립트의 트리거 구성에 대해 설명합니다.

- 트리거 유형: **페이지 보기**
- 이 트리거는 다음에서 발생합니다: **일부 페이지 보기**
- 이 트리거 조건을 발생시킬 조건은: **페이지 URL이 지정된 페이지를 포함하지 않습니다. 예: (index/index/category/printers)** (아래 그림 참조)

![](https://imagedelivery.net/5m26Aj-CutMXPPNacMs_yQ/3c40253f-ed9f-4943-9edd-94be3ca49a00/public)

3. 그런 다음 태그로 이동하여 구성한 트리거를 선택합니다(아래 그림 참조).
![](https://imagedelivery.net/5m26Aj-CutMXPPNacMs_yQ/63334e93-92ed-4892-7b14-70d601048300/public)

### 메신저 + 팝업 스크립트에서 트리거 구성:
지정된 페이지를 포함하지 않는 메신저 스크립트의 경우, 메신저 스크립트와 팝업 스크립트를 구성해야 합니다. 다음 단계를 따르세요.

1. 구글 태그 관리자 코드에서는 메신저와 팝업 코드 모두를 붙여넣어야 합니다(아래 그림 참조).
![](https://imagedelivery.net/5m26Aj-CutMXPPNacMs_yQ/319dd0cd-f4e9-4b24-c1b0-8a833982d700/public)

2. HTML 코드에 트리거를 추가합니다. 코드 트리거 조건에는 지정된 페이지가 포함됩니다.

- 트리거 유형: **페이지 보기**
- 이 트리거는 다음에서 발생합니다: **일부 페이지 보기**
- 이 트리거 조건을 발생시킬 조건은: **페이지 URL이 지정된 페이지를 포함합니다. 예: (index/index/category/printers)** (아래 그림 참조).

![](https://imagedelivery.net/5m26Aj-CutMXPPNacMs_yQ/9c88c4c6-bcdd-404c-6588-02d97c07cf00/public)

3. 그런 다음 태그로 이동하여 구성한 트리거를 선택합니다(아래 그림 참조).
![](https://imagedelivery.net/5m26Aj-CutMXPPNacMs_yQ/c0d1d632-65c9-4193-0834-c6a9b4d20600/public)
4. 트리거 구성과 함께 구글 태그 스크립트 그림을 참조하세요.
