# 코드베이스에 기여하기

erxes에 기여해주셔서 감사합니다! 이 문서는 이 저장소에 변경 사항을 제출하는 방법과 따를 규칙을 설명합니다. 궁금한 점이 있으면 여기에 이슈를 제출하거나 [Discord](https://discord.com/invite/aaGzy3gQK5)를 통해 문의해 주세요.

## 사전 요구사항

- GitHub 이슈 및 풀 리퀘스트에 익숙해야 합니다.
- 문서를 읽어야 합니다.
- erxes로 테스트 프로젝트를 설정해야 합니다.
## PR 이전에 이슈 생성

1. 변경 작업을 시작하기 전에 작업할 내용에 대한 이슈가 있는지 확인해 주세요. [기존 이슈](https://github.com/erxes/erxes/issues)를 찾거나 [새 이슈를 열어](https://github.com/erxes/erxes/issues/new/choose) 이슈가 없는 경우 열어 주세요. 이렇게 하면 다른 사람들이 의견을 제시하거나 대안을 제안할 수 있어 erxes의 미래에 가장 적합한 변경만 추가되도록 할 수 있습니다.
2. 변경 작업을 시작할 준비가 되면 먼저 [erxes 저장소를 포크](https://docs.github.com/en/get-started/quickstart/fork-a-repo)하고 [브랜치를 생성](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/proposing-changes-to-your-work-with-pull-requests/creating-and-deleting-branches-within-your-repository)하세요. develop 브랜치에서 시작하세요.
3. 변경 작업을 수행하세요.
4. [erxes 저장소의 develop 브랜치를 대상으로 풀 리퀘스트를 생성](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/proposing-changes-to-your-work-with-pull-requests/creating-a-pull-request-from-a-fork)하세요. 며칠 내에 erxes 팀원이 PR을 검토하고 의견을 제시하며 최종적으로 승인할 것입니다.
# 작업 흐름
### 브랜치

모든 변경 사항은 브랜치의 일부여야 하며 풀 리퀘스트로 제출해야 합니다. 브랜치는 다음 중 하나로 접두어를 붙여야 합니다:

- fix/ - 버그 수정
- feat/ - 기능 추가
- docs/ - 문서 변경

### 커밋

커밋은 작고 독립적으로 유지하려고 노력하세요. 이렇게 하면 리뷰어가 내용을 이해하기 쉽고 요청을 처리하기 쉬워집니다.

### 풀 리퀘스트

변경 작업이 완료되면 브랜치를 풀 리퀘스트로 제출해야 합니다. 풀 리퀘스트는 erxes 저장소의 develop 브랜치를 대상으로 열어야 합니다.

PR 설명에는 다음 구조를 따라야 합니다:

- 무엇 - 이 PR에 포함된 변경 사항
- 왜 - 이러한 변경이 왜 중요한지
- 어떻게 - 변경 사항이 어떻게 구현되었는지
- 테스트 - 변경 사항이 어떻게 테스트되었는지 또는 리뷰어가 기능을 테스트하는 방법

리뷰 요청 전에 자체 검토를 강력히 권장합니다. 자체 검토를 하려면 오른쪽 상단의 검토 버튼을 클릭하고 코드를 검토하여 변경 사항에 주석을 달아주세요. 이렇게 하면 리뷰어가 PR을 처리하기 쉬워집니다.

### 병합 스타일

모든 풀 리퀘스트는 스쿼시(Squash) 병합됩니다.

### 테스트

모든 PR에는 포함된 변경 사항에 대한 테스트가 포함되어야 합니다. 우리는 다음 두 가지 유형의 테스트를 작성해야 합니다:

- packages/*/src/services/**tests** 및 packages/*/src/api/routes/*/**tests**에 있는 단위 테스트
- integration-tests/*/**tests**에 있는 통합 테스트

# 문서화

- 일반적으로 코드에 주석을 추가하여 변경 사항을 문서화하는 것을 권장합니다.
- 사용자에게 영향을 미치는 동작을 변경하는 경우 해당 변경 사항에 대한 문서를 제공해야 합니다.
- 모든 메서드 및 엔드포인트는 [JSDoc](https://jsdoc.app/) 및 [swagger-inline](https://www.npmjs.com/package/swagger-inline)을 사용하여 문서화해야 합니다.

<Note>
  나중에 erxes 코드베이스에 대한 변경 사항을 문서화하는 데 기여하는 경우, 
  문서 웹사이트의 기여 지침도 확인하세요.
</Note>
