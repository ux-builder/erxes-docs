# 메신저 기능 조작하기

메신저는 특정 사용자 상호작용에서 프로그래밍 방식으로 실행될 수 있습니다. 예를 들어 "도움 요청 버튼 클릭" 같은 경우입니다. 문서에 첨부하는 대신, erxes 런처를 특정 요소에 첨부할 수 있는 옵션이 있어야 합니다. 이는 단일 페이지 앱에서 모든 페이지에 런처 아이콘을 표시하는 것이 이상적이지 않기 때문입니다. 이제 메신저 스타일, 위치 변경 및 특정 페이지에서 메신저를 호출하는 버튼 설정과 같은 메신저 기능을 조작할 수 있습니다.

**버튼 제출**
외부(부모 웹사이트)에서 제출을 호출하는 기능이 있습니다. 이는 외부에서 callSubmit 액션을 수신하여 강제로 제출 액션을 수행하는 것을 의미합니다. 예를 들어, 메신저를 열기 위해 액션을 호출하는 버튼을 추가할 수 있습니다. window.Erxes.showMessenger() 함수가 window 객체에서 사용할 수 있습니다. 이 함수를 사용하여 프로그래밍 방식으로 메신저를 표시할 수 있습니다. 다음 스크립트를 메신저 스크립트 코드에 삽입하십시오.

```tsx
 document.getElementById('button').onclick = () => {
    window.Erxes.showMessenger()
  }
  ```
먼저 기본 핸들러를 숨기기 위해 show launcher 체크를 해제해야 합니다. show launcher를 체크하면 위젯 섹션이 보이지 않지만 callSubmit 액션을 통해 메신저를 호출할 수 있습니다.

# 구글 태그 관리자 문제 해결 가이드
지정된 페이지에 구글 태그 관리자 스크립트를 설치할 때 문제가 발생하면, 설정이 올바르게 구성되었는지 확인하기 위한 몇 가지 문제 해결 단계를 따를 수 있습니다. 이 지침은 **모든 웹 페이지에 메신저가 포함되고 특정 페이지에만 팝업 스크립트가 포함된 경우**를 다룹니다. 따라서 다른 경우에도 erxes 스크립트를 변경하여 적용할 수 있습니다. 다음 지침은 웹 페이지에 erxes 스크립트를 배치하는 방법을 보여줍니다.

페이지에 두 개의 스크립트 코드를 설치하려면 구글 태그 관리자 HTML 코드에 일부 트리거를 설정해야 합니다.

### 메신저 스크립트에 트리거 구성:
1. 구글 태그 관리자에 이미 구성된 HTML 코드에 메신저 스크립트 트리거를 추가합니다.
2. 메신저 스크립트가 지정된 페이지를 포함하지 않도록 트리거를 구성합니다. 이 경우, 메신저 스크립트는 다른 태그에 구성됩니다. 다음 단락의 메신저 + 팝업 스크립트 트리거 구성 참조.

- 트리거 유형: **페이지 보기**
- 이 트리거가 실행되는 페이지: **일부 페이지 보기**
- 트리거 조건: **페이지 URL이 지정된 페이지를 포함하지 않음 (예: index/index/category/printers)** (아래 그림 참조)

![](https://imagedelivery.net/5m26Aj-CutMXPPNacMs_yQ/3c40253f-ed9f-4943-9edd-94be3ca49a00/public)

3. 그런 다음 태그로 이동하여 구성한 트리거를 선택합니다 (아래 그림 참조).
![](https://imagedelivery.net/5m26Aj-CutMXPPNacMs_yQ/63334e93-92ed-4892-7b14-70d601048300/public)

### 메신저 + 팝업 스크립트에 트리거 구성:
위의 경우와 같이 메신저 스크립트가 지정된 페이지를 포함하지 않는 경우, 메신저 스크립트와 팝업 스크립트를 함께 구성해야 합니다. 다음 단계를 따릅니다.

1. 구글 태그 관리자 코드에 메신저와 팝업 코드의 스크립트를 모두 붙여넣습니다 (아래 그림 참조).
![](https://imagedelivery.net/5m26Aj-CutMXPPNacMs_yQ/319dd0cd-f4e9-4b24-c1b0-8a833982d700/public)

2. HTML 코드에 트리거를 추가합니다. 트리거 조건은 지정된 페이지를 포함해야 합니다.

- 트리거 유형: **페이지 보기**
- 이 트리거가 실행되는 페이지: **일부 페이지 보기**
- 트리거 조건: **페이지 URL이 지정된 페이지를 포함 (예: index/index/category/printers)** (아래 그림 참조)

![](https://imagedelivery.net/5m26Aj-CutMXPPNacMs_yQ/9c88c4c6-bcdd-404c-6588-02d97c07cf00/public)

3. 그런 다음 태그로 이동하여 구성한 트리거를 선택합니다 (아래 그림 참조).
![](https://imagedelivery.net/5m26Aj-CutMXPPNacMs_yQ/c0d1d632-65c9-4193-0834-c6a9b4d20600/public)
4. 구글 태그 관리자 스크립트와 트리거 구성에 대한 그림을 참조합니다.
