# 오류

이 가이드에서는 API 작업 중에 문제가 발생했을 때 어떤 일이 일어나는지 설명합니다. 실수는 일어나기 마련이며, 대부분은 우리의 실수가 아니라 여러분의 실수일 가능성이 큽니다. 자주 마주치게 될 상태 코드와 오류 유형을 살펴보겠습니다. {{ className: 'lead' }}

API 응답을 받을 때 상태 코드를 확인하여 요청이 성공했는지 여부를 알 수 있습니다. 응답이 실패하면 오류 유형과 오류 메시지를 사용하여 무엇이 잘못되었는지 파악하고 간단한 디버깅을 수행할 수 있습니다(지원 팀에 문의하기 전에).

<Note>
  오류에 대해 지원 팀에 문의하기 전에 모든 보고된 오류의 99%가 사실은 사용자 오류임을 인지하시기 바랍니다. 
  그러므로 지원 팀에 연락하기 전에 코드를 꼼꼼히 확인해 주세요.

---

## 상태 코드

다음은 Protocol API에서 반환되는 다양한 상태 코드의 범주 목록입니다. 이를 사용하여 요청이 성공했는지 여부를 이해할 수 있습니다.

<Properties>
  <Property name="2xx">
    2xx 상태 코드는 성공적인 응답을 나타냅니다.
  </Property>
  <Property name="4xx">
    4xx 상태 코드는 클라이언트 오류를 나타내며, 이는 _여러분_ 의 문제입니다.
  </Property>
  <Property name="5xx">
    5xx 상태 코드는 서버 오류를 나타내며, 이러한 오류는 거의 발생하지 않습니다.
  </Property>
</Properties>

## 오류 유형

<Row>
  <Col>

    요청이 실패할 때마다 Protocol API는 오류 유형과 메시지가 포함된 오류 응답을 반환합니다. 이 정보를 사용하여 무엇이 잘못되었는지 더 잘 이해하고 해결 방법을 찾을 수 있습니다. 대부분의 오류 메시지는 매우 도움이 되고 실행 가능한 내용입니다.

    다음은 Protocol API에서 지원하는 두 가지 오류 유형 목록입니다. 이를 사용하여 여러분이 무엇을 잘못했는지 이해할 수 있습니다.

    <Properties>

      </Property>
      <Property name="invalid_request">
        이는 여러분이 실수한 경우를 의미하며, 훨씬 더 가능성이 높습니다.
      </Property>
    </Properties>

  </Col>
